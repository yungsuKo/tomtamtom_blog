# 반응형 웹을 만드는 방법론

다양한 디바이스와 디바이스의 크기에 대응하는 디자인 접근방식이다.

## 미디어 쿼리

미디어 쿼리를 사용하는 경우 일련의 테스트(ex: 사용자의 화면이 특정 너비 또는 특정 해상도보다 큰지 여부)를 실행하고 css를 선택적으로 적용하여 사용자의 요구에 맞게 페이지의 스타일을 지정할 수 있다.  
예를 들어 아래 미디어 쿼리 css의 경우 웹페이지가 화면 미디어로 표시되고 있고, 뷰포트의 너비가 80rem 이상인지 테스트한다.  
그리고 참인 경우 .container 클래스에 해당 css를 적용한다.

```css
@media screen and (min-width: 80rem) {
  .container {
    margin: 1em 2em;
  }
}
```

이와 같이 분기점을 사용하는 경우 개별 디바이스의 절대적인 크기보다는 css 값과 단위를 사용하여 미디어 쿼리의 분기점을 정의하는 것이 좋다.

## 유동형 그리드

그리드를 기반으로 구축되므로 모든 기기 크기를 타겟팅할 필요가 없다.
분기점을 추가하여 콘텐츠가 보기 좋지 않게 보이는 지점에서 디자인을 변경할 수 있다.
그리드는 웹 화면의 폭을 몇 개의 컬럼으로 나누는 것이다.
상황에 따라서 몇 개의 컬럼들을 묶어서 사용하며, 컬럼들의 폭과 다른 컬럼과의 간격도 일정하게 유지 한다.
유동형 그리드는 그리드의 폭을 고정 값이 아닌 em 또는 %의 값으로 설정하는 것을 말한다.
즉, 가로 폭길이의 변화에 따라서 컬럼의 크기가 상대적으로 변하는 것이다.

## 유동형 레이아웃

레이아웃 크기를 유동형 그리드와 같이 상대적 단위로 지정하여 웹의 크기에 따라 유동적으로 변화를 준다.
미디어 쿼리를 사용하여 일정 크기가 되면 레이아웃 구조를 바꾸어 주는 방법이다.
특정 부분이 사라질 수도 있고, 합쳐지기도 하면서 유동적으로 레이아웃 구조가 바뀌며 사용자에게 제공된다.
